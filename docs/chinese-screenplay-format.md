# 中国电视剧标准剧本格式规范

> 学习来源：示例剧本《时光之桥》
> 用途：作为 Stage 4 AI 生成提示词和 Stage 5 格式化输出的参考标准

---

## 一、剧本整体结构

```
标题：《作品名》
第一幕：幕标题（场景X —— 地点描述，时间）
[场景标题行]

【动作/镜头描述】

角色介绍：（可选，首次出场时）

角色名： （表情/动作提示）台词内容

旁白（声音描述）： 旁白内容

角色名（内心独白）： 内心独白内容

[转换场景：描述]

...

剧终
```

---

## 二、各类元素的格式规范

### 2.1 幕标题
```
第X幕：幕名称（场景X —— 地点简述，时间）
```
- 示例：`第一幕：开篇（场景一 —— 现代都市，夜晚）`
- 说明：每幕开头，概括本幕的场景编号、地点、时间

### 2.2 场景标题 (scene_heading)
```
[外景/内景：地点详细描述，时间]
```
- 示例：`[外景：繁华的都市夜景，霓虹闪烁]`
- 示例：`[转换场景：时间回溯至1935年，同一地点的旧式庭院]`
- 格式：方括号包裹，内外景标识 + 地点 + 时间

### 2.3 动作/镜头描述 (action)
```
【描述文字】
```
- 示例：`【镜头缓缓推进，高楼林立的城市中，一辆出租车穿梭在灯火阑珊的街道上。】`
- 示例：`【车门打开，主角林浅（女，28岁，现代都市白领）走出车外，抬头望向远处的一栋老建筑，眼中闪烁着复杂的情绪。】`
- 示例：`【镜头切换为柔和的复古色调，展现一座充满历史感的庭院，阳光透过树叶洒在青石板路上。】`
- 说明：用【】包裹，描述镜头运动、场景环境、角色动作、表情神态
- 角色首次出场时在此标注全名、性别、年龄、身份

### 2.4 角色对白 (dialogue)
```
角色名： （表情/动作提示）台词内容
```
- 示例：`方晴： （轻声细语）又在想你的新小说了吗？别太累了。`
- 示例：`苏墨： （微笑中带着一丝愁绪）我在构思一个关于时空交错的故事，想让两个不同时代的人相遇。`
- 示例：`林浅： （急切地）您知道这位叫苏墨的人吗？他的故事似乎与我有着不解之缘。`
- 格式：`角色名：` + 空格 + `（表情/语气提示）` + 台词
- 无表情提示时可以省略括号：`林浅： （自言自语）原来这里曾是文学社的活动地点……`

### 2.5 旁白 (narration)
```
旁白（声音描述）： 旁白内容
```
- 示例：`旁白（温暖而深沉的男声）： 在这个快节奏的时代，每个人都是自己故事的主角，却也常常成为他人生命中的过客。但总有一些瞬间，能跨越时空，连接起看似无关的灵魂。`
- 格式：`旁白` + `（声音质感描述）` + `：` + 内容
- 用于故事背景介绍、过渡衔接、情感升华

### 2.6 内心独白 (inner_monologue)
```
角色名（内心独白）： 内心内容
```
- 示例：`林浅（内心独白）： 这是……怎么回事？为什么我会感到如此熟悉？`
- 格式：`角色名` + `（内心独白）` + `：` + 内容
- 用于表达角色内心想法，与对白区分

### 2.7 角色介绍 (character_intro)
```
角色介绍：

角色名（性别，年龄，身份/背景）
角色名（性别，年龄，身份/背景）
```
- 示例：
```
角色介绍：

苏墨（男，25岁，那个时代的知识青年）
方晴（女，24岁，苏墨的青梅竹马，温柔善良）
```
- 说明：在角色首次集中出场时，单独列出角色介绍段落

### 2.8 转场 (transition)
```
[转换场景：描述]
```
- 示例：`[转换场景：时间回溯至1935年，同一地点的旧式庭院]`
- 配合镜头切换描述使用

### 2.9 剧终标记
```
剧终
```
- 全剧结束的标志

## 三、AI 生成 JSON 输出格式（Stage 4）

基于以上格式规范，AI 在 Stage 4 应输出以下 JSON 结构：

```json
{
  "scene_heading": "[外景：繁华都市夜景，夜晚]",
  "content": [
    {
      "type": "action",
      "text": "镜头缓缓推进，高楼林立的城市中，一辆出租车穿梭在灯火阑珊的街道上。"
    },
    {
      "type": "narration",
      "text": "在这个快节奏的时代，每个人都是自己故事的主角，却也常常成为他人生命中的过客。",
      "direction": "温暖而深沉的男声"
    },
    {
      "type": "action",
      "text": "车门打开，林浅走出车外，抬头望向远处的一栋老建筑，眼中闪烁着复杂的情绪。"
    },
    {
      "type": "dialogue",
      "character": "方晴",
      "text": "又在想你的新小说了吗？别太累了。",
      "direction": "轻声细语"
    },
    {
      "type": "dialogue",
      "character": "苏墨",
      "text": "我在构思一个关于时空交错的故事，想让两个不同时代的人相遇。",
      "direction": "微笑中带着一丝愁绪"
    },
    {
      "type": "inner_monologue",
      "character": "林浅",
      "text": "这是……怎么回事？为什么我会感到如此熟悉？"
    },
    {
      "type": "transition",
      "text": "时光流转，回到现代。"
    }
  ]
}
```

### 内容类型一览

| type | 含义 | 必填字段 | 可选字段 | 对应剧本格式 |
|------|------|----------|----------|-------------|
| `action` | 动作/镜头描述 | `text` | - | `【text】` |
| `dialogue` | 角色对白 | `character`, `text` | `direction` | `角色名： （direction）text` |
| `narration` | 旁白 | `text` | `direction` | `旁白（direction）： text` |
| `inner_monologue` | 内心独白 | `character`, `text` | - | `角色名（内心独白）： text` |
| `transition` | 转场 | `text` | - | `[转换场景：text]` |
| `note` | 备注/提示 | `text` | - | 剧本注释（不显示在正文中） |

## 四、YAML 输出格式（Stage 5 组装）

Stage 5 将结构化内容渲染为可读文本，存储在每个场景的 `scene_text` 字段中：

```yaml
scenes:
  - id: scene_001
    act: 1
    scene_heading: "[外景：繁华都市夜景，夜晚]"
    setting:
      location: 现代都市
      time: 夜晚
    characters_present: ["char_001", "char_002"]
    content:
      - type: action
        text: "镜头缓缓推进..."
      - type: narration
        text: "在这个快节奏的时代..."
        direction: "温暖而深沉的男声"
      - type: dialogue
        character: "char_002"
        text: "又在想你的新小说了吗？"
        direction: "轻声细语"
    scene_text: |
      [外景：繁华都市夜景，夜晚]

      【镜头缓缓推进，高楼林立的城市中，一辆出租车穿梭在灯火阑珊的街道上。】

      旁白（温暖而深沉的男声）： 在这个快节奏的时代，每个人都是自己故事的主角。

      方晴： （轻声细语）又在想你的新小说了吗？别太累了。
```

- `content` 保留结构化数据（机器可读）
- `scene_text` 是格式化后的可读剧本文本（人类可读）
- 两者并存，互不干扰
